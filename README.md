# Song Recommendation System

Content-based song recommender (cosine similarity over Spotify audio features + genre,
with MMR diversity re-ranking). Built from `notebook/song_recommendation.ipynb`.

## Project structure

```
song-recommendation-system/
├── app/                 FastAPI backend
│   ├── main.py           app entrypoint (uvicorn app.main:app)
│   ├── router.py         /search, /recommend/{id}, /genres, /health
│   ├── state.py          loads model artifacts once at startup
│   └── schemas.py        request/response models
├── src/                  ML pipeline modules (reusable, imported by app + main.py)
│   ├── data_loader.py
│   ├── preprocessing.py
│   ├── features.py
│   ├── recommender.py    recommend_songs (baseline) + recommend_songs_mmr (default)
│   ├── evaluation.py
│   └── persistence.py    save/load trained artifacts
├── frontend/
│   └── index.html         single-file UI: search a song, see recommendations
├── models/                saved artifacts (.pkl, .json) — generated by main.py
├── data/                  dataset.csv (raw, not tracked in git ideally)
├── notebook/               original exploratory notebook (EDA lives here only)
├── main.py                 full pipeline runner: load → clean → features → eval → save
├── Dockerfile
└── requirements.txt
```

## Setup

```bash
pip install -r requirements.txt
```

## 1. Run the ML pipeline

Builds the model and writes everything the API needs into `models/`:

```bash
python main.py
```

Output: `models/feature_matrix.pkl`, `scaler.pkl`, `feature_columns.json`,
`songs_lookup.pkl`, `model_metadata.json`.

## 2. Run the API

```bash
uvicorn app.main:app --reload
```

Docs at `http://localhost:8000/docs`. Endpoints:

| Endpoint | Description |
|---|---|
| `GET /health` | model status + eval metrics |
| `GET /search?q=love&limit=20` | search by track name / artist |
| `GET /recommend/{track_id}?n=10&method=mmr` | get recommendations (`method`: `mmr` or `baseline`) |
| `GET /genres` | list all genres in the catalog |

## 3. Run the frontend

`frontend/index.html` is a single static file — just open it in a browser
(or serve it with any static server). It calls the API at `http://localhost:8000`
by default; edit `API_BASE` at the top of the `<script>` tag if you deploy the
backend elsewhere.

## Docker

```bash
python main.py                 # generate models/ first, on the host
docker build -t song-rec .
docker run -p 8000:8000 song-rec
```

## Notes

- No collaborative filtering: this dataset has no user/ratings data, only
  track-level audio features, so a content-based approach is the correct fit.
- `mmr` is the default recommender — better catalog coverage and intra-list
  diversity than plain top-N, at a modest genre-precision cost. See
  `model_metadata.json` → `eval_summary` after running the pipeline, or the
  notebook's Evaluation section for the full writeup.
